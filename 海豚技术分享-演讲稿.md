## 开场：一个周六下午的电话

大家好，我是陈辉。在正式开始之前，我想先跟大家分享一个小故事。

就在去年的一个周六下午，我正窝在沙发上准备好好休息一下，手机突然响了——是边江打来的。他语气很急："辉哥，系统出问题了，表单提交不了，好几个工单卡在那儿，业务那边催得紧。"我放下手里的水杯，打开电脑，远程排查、定位问题、修复验证，一套流程走下来，大半个下午就没了。

我相信在座的很多同学都有类似的经历——**周末、休息时间，依然在处理各种运维工作**。大量的运维事务占用了我们休息的时间，重复的工单、权限申请、问题排查……日复一日。那：**有没有一种方式，可以把我们从这种循环中解放出来？**

这个问题，就是今天分享的起点——**运维智能体的诞生**。

## 前言：运维智能体的诞生

在讲具体方案之前，我们先直面运维的三个核心痛点：

- **周末仍在处理**——周末、休息时间依然在处理各种运维工作，就像我刚才说的那个周六下午；
- **大量时间占用**——大量的运维占用的时间长，一天下来感觉都在"救火"；
- **如何破冰？**——被各种运维占用了所有的时间，怎么破冰？

带着这个问题，我们开始了行动，思路很清晰：

1. **建立知识库**作为基石——让经验可沉淀、可复用；
2. **接入珊瑚系统**来做 AI 分析——让 AI 能理解运维场景；
3. **在 PC 端嵌入一个悬浮框**来进行对话——让用户随时随地能和 AI 交互。

初期效果很直观：**通过知识库 + AI 分析 + PC 悬浮对话，初步实现了运维智能化**。这就是海豚的雏形。

## 一、海豚桌面端——桌面端展示、运维痛点与技术选型

### 1.1 为什么需要海豚？

运维的痛点其实可以归纳为四个方面：

- **重复性事务**——大量工单、权限申请等重复操作，消耗运维人力，效率低下。就拿网络权限申请来说，每次都要填一堆字段，流程一模一样，但每次都得手动来一遍。
- **经验依赖**——问题排查依赖个人经验，知识难以沉淀和传承。老员工离职了，经验也跟着走了。
- **信息孤岛**——缺乏统一入口，各系统割裂，信息分散。运维要来回切换好几个系统才能完成一件事。
- **AI 难触达**——AI 能力无法以交互式界面直接触达用户。AI 只能输出文字，用户没法直接在 AI 的回复里完成操作。

而海豚给出的答案是四个关键词：**统一入口**——一个应用覆盖运维全场景；**智能交互**——自然语言驱动，降低门槛；**A2UI 赋能**——AI 动态生成界面；**高效协作**——会话全流程线上化。

### 1.2 技术选型与核心功能

海豚桌面端基于 **Electron 41** 构建，技术栈选型如下：

| 层面 | 选型 |
|------|------|
| 前端框架 | Vue 3 + Vite 6 |
| UI 组件库 | Element Plus |
| 桌面端 | Electron 41 |
| 动态渲染 | A2UI Vue Engine |
| 数据流 | 珊瑚系统 → JSON Schema |

核心功能覆盖四个方面：

- **智能聊天**——AI 实时对话 + A2UI 表单嵌入，对话中直接生成可操作的界面；
- **会话管理**——全生命周期管理 + 状态追踪，每个运维会话从创建到结束全程可追溯；
- **悬浮球与托盘**——置顶悬浮球 + 系统托盘集成，随时唤起，不打断工作流；
- **A2UI 动态渲染**——JSON Schema 驱动 + 零代码扩展，这是海豚最核心的能力。

海豚桌面端就是基于 Electron 41 构建的运维智能助手桌面应用，它把 AI 能力直接带到了用户的桌面上。

## 二、A2UI——从需求中诞生的方案

### 2.1 什么是 A2UI？

刚才提到海豚最核心的能力是 A2UI，那 A2UI 到底是什么？八个字概括：**对话即界面，界面即对话**。

- **对话即界面**——用户通过自然语言与 AI 对话，AI 实时生成可交互的 UI 界面。你不需要去某个系统里找表单，你只需要说"我要申请网络权限"，界面就出来了。
- **界面即对话**——界面上的交互操作直接反馈到 AI 对话流中，形成闭环。你在界面上填了什么、选了什么，AI 都知道，对话不会断。

这个理念其实最早来自 Google 的愿景——让 AI 不只是输出文字，而是直接输出可交互的界面。我们把这个愿景落地到了企业级场景中，形成了 A2UI。

### 2.2 阶段性成果

光说理念可能比较抽象，我们直接来看 A2UI 在实际运维场景中的阶段性成果，分三个阶段：

**阶段一：输入一段话自动生成表单**

这是我们最开始的。用户只需要在对话框里输入一句话，比如"给我生成一个报销单"，AI 就能理解意图，自动生成对应的表单界面。用户直接在对话中填写、提交，不需要跳转到任何其他系统。

这就是"对话即界面"的最直接体现——**你说什么，界面就生成什么**。

**阶段二：根据提示词生成——网络权限申请**

这是我们目前的阶段-根据提示词生成界面。比如说：网络权限申请；这个场景涉及大量字段——上网权限、IM 权限、邮箱权限、USB 权限、VPN 权限、申请原因等等，每个字段还有不同的取值范围。

传统方式下，用户需要登录 OA 系统，找到对应的流程，一个字段一个字段地填写，很容易填错。而现在我们只需要使用接口卡片，利用接口卡片中的入参生成提示词，再根据提示词自动生成对应的网络权限申请界面。

**阶段三（规划）：根据 SKILLS 生成**

未来我们规划了第三阶段——根据 SKILLS 来生成界面。这将进一步降低使用门槛，让运维工作真正做到"所说即所得"。

## 三、A2UI 实现——技术架构与渲染机制

前面我们看到了 A2UI 能做什么，那它是怎么做到的？这一部分我们来揭开 A2UI 的技术实现。

### 3.1 A2UI 与大模型协同

大模型赋予了机器理解与生成的能力，但有一个关键缺失——**缺乏与用户交互的界面载体**。大模型只能输出文本，用户看到文本后还得自己去系统里操作，这个断层就是效率的瓶颈。

A2UI 正是连接大模型与用户的桥梁，让 AI 的输出不再局限于文本，而是直接成为可交互的界面。它实现了三个关键能力：

- **规范化输出**——大模型按统一 Schema 生成 UI，确保一致性；
- **即时渲染**——Schema 到界面毫秒级转化，零延迟响应；
- **业务适配**——组件可扩展，灵活适配不同业务场景。

### 3.2 JSON Schema 驱动的 UI 渲染机制

A2UI 的渲染流程非常清晰，四步完成：

1. **珊瑚系统输出 JSON Schema**——大模型理解用户意图后，输出结构化的 UI 描述；
2. **A2UIRoot 解析为树形结构**——将 JSON Schema 解析成组件树；
3. **Renderer 映射为 Vue VNodes**——每个节点映射到对应的 Vue 组件；
4. **用户交互触发 Action 事件**——用户操作反馈到对话流中。

举个例子，一段简单的 Schema：

```json
{
  "type": "Card",
  "props": { "title": "运维工单" },
  "children": [
    { "type": "TextField", "props": { "label": "问题描述", "field": "desc" } },
    { "type": "Button", "props": { "label": "提交", "action": "submit" } }
  ]
}
```

Schema 到 UI 的对应关系一目了然：`"type": "Card"` 渲染为卡片容器，`"type": "TextField"` 渲染为文本输入框，`"type": "Button"` 渲染为操作按钮，`"props.field"` 做数据绑定，`"props.action"` 处理点击事件。

### 3.3 A2UI 组件概览与核心能力

A2UI 提供了统一规范驱动的组件体系，覆盖表单、展示、交互等全场景组件，赋能大模型高效生成前端 UI。每个组件都有明确的 type 标识和 props 定义，大模型只需要按照规范输出 JSON，就能生成完整可交互的界面。

### 3.4 组件库开源

为了以后各个系统都可以接入 A2UI，我已经将它封装为独立的 **npm 组件库**并发布，任何项目都可以通过 `npm install` 直接引入，无需重复开发。它具备四个特点：

- **开箱即用**——npm install 即可引入；
- **版本管控**——语义化版本，升级可控；
- **跨项目复用**——一次开发，多系统复用；
- **生态开放**——开源可扩展，社区共建。

## 总结

回到开头那个周六下午的故事。如果当时有海豚，边江只需要在对话里里说一句"国内售后系统，工单中心，维修单提交不了"，AI 就能自动定位问题、生成修复方案，甚至直接在对话中完成操作。我不需要放下手里的水杯，他也不需要打那个电话。

今天我分享了三个部分：

1. **海豚桌面端**——从运维痛点出发，用 Electron + Vue 3 + A2UI 构建了运维智能助手的桌面应用，统一入口、智能交互、高效协作；
2. **A2UI 方案与阶段性成果**——"对话即界面，界面即对话"的理念，并通过三个阶段性成果，展示了从自动生成表单到网络权限申请再到未来 SKILLS 驱动的演进路径；
3. **A2UI 实现**——JSON Schema 驱动的渲染机制、与大模型的协同方式、组件体系，以及 npm 组件库的开源发布，让 AI 的输出从文本升级为可交互的界面。

**运维不应该占用我们所有的时间。海豚和 A2UI，就是为了让我们的时间回归到更有价值的事情上。**
